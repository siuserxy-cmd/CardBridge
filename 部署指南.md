## 🚂 方案一：Railway（最推荐）

### 为什么推荐？
- ✅ 完全免费（有额度限制）
- ✅ 自动部署
- ✅ 支持数据库持久化
- ✅ 提供免费域名

### 部署步骤

1. **准备 GitHub 仓库**
   ```bash
   # 在项目文件夹运行
   git init
   git add .
   git commit -m "Initial commit"

   # 在 GitHub 创建新仓库后
   git remote add origin https://github.com/你的用户名/仓库名.git
   git push -u origin main
   ```

2. **部署到 Railway**
   - 访问 https://railway.app
   - 点击 "Start a New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择你的仓库
   - 自动检测 Node.js 项目

3. **配置环境变量**
   在 Railway 项目设置中添加：
   ```
   NODE_ENV=production
   PORT=3000
   JWT_SECRET=你的随机密钥（32位以上）
   ADMIN_EMAIL=admin@example.com
   ADMIN_PASSWORD=你的管理员密码
   ```

4. **等待部署完成**
   - Railway 会自动安装依赖
   - 自动初始化数据库
   - 提供一个公开网址

5. **访问网站**
   - 复制 Railway 提供的网址
   - 访问 `你的网址.railway.app`
   - 完成！🎉

## 🎨 方案二：Render

### 部署步骤

1. **上传代码到 GitHub**（同上）

2. **创建 Render 服务**
   - 访问 https://render.com
   - 点击 "New +" → "Web Service"
   - 连接你的 GitHub 仓库

3. **配置服务**
   - **Name**: digital-shop
   - **Environment**: Node
   - **Build Command**: `npm install && npm run init-db`
   - **Start Command**: `npm start`

4. **添加环境变量**
   在 Environment 标签添加：
   ```
   NODE_ENV=production
   JWT_SECRET=你的随机密钥
   ADMIN_EMAIL=admin@example.com
   ADMIN_PASSWORD=你的密码
   ```

5. **部署**
   - 点击 "Create Web Service"
   - 等待部署完成
   - 获得免费域名：`你的项目.onrender.com`

## 🐳 方案三：Docker 部署（适合自己的服务器）

### 前提条件
- 已安装 Docker 和 Docker Compose
- 有自己的服务器（阿里云、腾讯云等）

### 快速启动

```bash
# 1. 在服务器上克隆代码
git clone 你的仓库地址
cd 发卡网

# 2. 修改 docker-compose.yml 中的环境变量
vim docker-compose.yml
# 修改 JWT_SECRET 和 ADMIN_PASSWORD

# 3. 启动容器
docker-compose up -d

# 4. 查看日志
docker-compose logs -f

# 5. 访问
# http://你的服务器IP:3000
```

### 常用命令

```bash
# 查看容器状态
docker-compose ps

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f digital-shop
```

## 🌍 方案四：传统服务器部署

### 1. 购买云服务器
- 阿里云 ECS
- 腾讯云 CVM
- 推荐配置：1核2G，CentOS/Ubuntu

### 2. 安装环境

```bash
# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 PM2（进程管理器）
sudo npm install -g pm2
```

### 3. 上传代码

```bash
# 使用 Git
git clone 你的仓库地址
cd 发卡网

# 或者使用 SCP 上传
scp -r 本地路径/* root@服务器IP:/root/digital-shop/
```

### 4. 启动服务

```bash
# 安装依赖
npm install

# 初始化数据库
npm run init-db

# 添加示例卡密（可选）
npm run add-cards

# 使用 PM2 启动
pm2 start server/index.js --name digital-shop

# 保存 PM2 配置
pm2 save
pm2 startup
```

### 5. 配置 Nginx（可选，用于域名访问）

```nginx
# /etc/nginx/sites-available/digital-shop
server {
    listen 80;
    server_name 你的域名.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/digital-shop /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 6. 配置 HTTPS（推荐）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d 你的域名.com

# 自动续期
sudo certbot renew --dry-run
```

## 🔒 安全建议

### 部署前必做

1. **修改 JWT_SECRET**
   ```bash
   # 生成随机密钥
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

2. **修改管理员密码**
   - 不要使用默认密码 `admin123456`
   - 使用强密码（至少12位，包含大小写字母、数字、符号）

3. **配置环境变量**
   - 不要在代码中硬编码敏感信息
   - 使用平台的环境变量功能

4. **数据库备份**
   ```bash
   # 定期备份数据库文件
   cp database/shop.db database/backup/shop-$(date +%Y%m%d).db
   ```

5. **更新依赖**
   ```bash
   # 检查安全漏洞
   npm audit

   # 修复漏洞
   npm audit fix
   ```

## 💡 常见问题

### 1. 部署后无法访问？
- 检查防火墙是否开放 3000 端口
- 检查环境变量是否正确配置

### 2. 数据库文件丢失？
- Railway/Render 免费版重启会清空数据
- 需要升级到付费版或使用外部数据库

### 3. 如何绑定域名？
- 在域名服务商添加 CNAME 记录
- 指向部署平台提供的域名

### 4. 支付功能无法使用？
- 确保配置了支付宝/微信的商户信息
- 测试模式下会模拟支付成功
